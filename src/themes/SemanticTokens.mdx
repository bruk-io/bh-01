# Default Theme

This is the **reference implementation** â€” the opinionated mapping that ships with `@bruk-io/bh-01`.
It wires [semantic token names](/docs/tokens-reference--docs) to primitives for light and dark modes.

To create your own theme, provide your own values for every token listed in the [Token Reference](/docs/tokens-reference--docs).

## How It Works

The default theme imports all primitives, then maps them to semantic tokens for light and dark mode.

```css
/* src/themes/default.css */
@import '../tokens/index.css';   /* all primitives */

:root, [data-theme='light'] {
  --bh-color-surface:        var(--bh-color-pearl);
  --bh-color-primary:        var(--bh-color-mandarin);
  --bh-color-danger:         var(--bh-color-amaranth);
  /* ... */
}

[data-theme='dark'] {
  --bh-color-surface:        var(--bh-color-thunder);
  --bh-color-primary:        var(--bh-color-mandarin);
  --bh-color-danger:         var(--bh-color-amaranth);
  /* ... */
}
```

Dark mode activates when `data-theme="dark"` is set on any ancestor element.
